ТЕХНИЧЕСКОЕ ЗАДАНИЕ

на разработку информационной системы
«Библиотечный каталог с системой учета и бронирования книг»

1. Общие сведения
1.1 Название проекта

Информационная система
«Библиотечный каталог»

1.2 Основание для разработки

Дипломная работа по профессии Backend-разработчик.

1.3 Цель разработки

Создание серверной части информационной системы для:

учета библиотечного фонда;

поиска книг;

выдачи и возврата книг;

простого бронирования;

фиксации утерянных и поступивших книг.

2. Назначение системы

Система предназначена для автоматизации работы библиотеки и упрощения взаимодействия между:

библиотекарем (администратором);

читателями.

Система должна обеспечивать:

централизованное хранение данных;

быстрый поиск книг;

контроль статуса каждой книги.

3. Функциональные требования
3.1 Роли пользователей
1) Администратор (библиотекарь)

добавляет и редактирует книги;

регистрирует пользователей;

выдает и принимает книги;

отмечает книги как утерянные;

просматривает статистику.

2) Пользователь (читатель)

ищет книги;

просматривает каталог;

бронирует книги;

видит список взятых книг и историю.

3.2 Функционал системы
3.2.1 Работа с книгами

добавление новой книги;

редактирование данных книги;

удаление книги;

учет количества экземпляров;

статус книги:

доступна;

забронирована;

выдана;

утеряна.

3.2.2 Поиск и фильтрация

Поиск книг по:

названию;

автору;

жанру;

году издания;

ISBN.

3.2.3 Выдача и возврат книг

оформление выдачи книги пользователю;

регистрация даты выдачи;

регистрация даты возврата;

автоматическое изменение статуса книги.

3.2.4 Бронирование

бронирование доступной книги;

ограничение по времени брони;

отмена бронирования;

автоматическое снятие брони по истечении срока.

3.2.5 Учет утерянных книг

пометка книги как утерянной;

исключение из доступных;

сохранение истории утери.

3.2.6 Учет поступивших книг

регистрация новых поступлений;

увеличение количества экземпляров;

сохранение даты поступления.

3.2.7 Учет пользователей

регистрация пользователей;

блокировка пользователей;

просмотр истории операций пользователя.

4. Нефункциональные требования
4.1 Требования к архитектуре

REST API;

клиент–серверная архитектура;

многослойная структура:

Controller;

Service;

Repository.

4.2 Требования к безопасности

аутентификация пользователей;

разграничение ролей;

защита от несанкционированного доступа.

4.3 Производительность

обработка запросов не более 2 секунд;

поддержка одновременной работы нескольких пользователей.

4.4 Надежность

сохранность данных при сбоях;

ведение журнала операций.

5. Требования к программной реализации
5.1 Язык и технологии

Язык программирования: Java

Framework: Spring Boot

ORM: Hibernate / JPA

База данных: PostgreSQL (или MySQL)

Формат обмена данными: JSON

Документация API: Swagger

6. Структура базы данных (основные сущности)

User (Пользователь)

Role (Роль)

Book (Книга)

Author (Автор)

Genre (Жанр)

BookCopy (Экземпляр книги)

Reservation (Бронирование)

Loan (Выдача)

LostBook (Утеря)

7. Входные и выходные данные
Входные данные

данные книг;

данные пользователей;

параметры поиска.

Выходные данные

список книг;

статус книг;

история выдач и бронирований;

статистические отчеты.

8. Ограничения и допущения

система не включает оплату штрафов;

пользовательский интерфейс может быть реализован отдельно;

основной акцент — на backend-логику.

9. Этапы разработки

Анализ предметной области

Проектирование БД

Разработка REST API

Реализация бизнес-логики

Тестирование

Документирование проекта

10. Ожидаемый результат

готовое backend-приложение;

REST API для работы с библиотекой;

документация и описание архитектуры;

демонстрация работы системы.
